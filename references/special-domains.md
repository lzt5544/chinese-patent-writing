# 特殊领域注意事项

## 软件/计算机程序类发明

根据《专利审查指南》第二部分第九章的规定：

**可授权主题：**
- 计算机程序本身（代码）不属于专利保护客体
- 但**包含计算机程序的技术方案**可以作为发明专利保护：工业过程控制、计算机内部运行性能改善、测量或测试过程控制、外部数据处理（图像处理、数据分析）等

**撰写要求：**
- 可布局为**方法权利要求**、**装置/功能模块权利要求**、**计算机设备/电子设备权利要求**、**计算机可读存储介质权利要求**或**计算机程序产品权利要求**；具体主题应与技术方案和保护重点匹配
- **装置权利要求**中，各组成部分应理解为**功能模块**（如"接收模块""处理模块""输出模块"），而非实体硬件
- 必须清楚描述：**输入数据 → 处理流程（含算法步骤） → 输出结果**的完整链条
- 涉及用户交互的，需描述用户操作与程序响应的对应关系

**权利要求撰写示例（方法）：**
```
一种基于XXX的数据处理方法，其特征在于，包括以下步骤：
S1：接收...数据；
S2：对所述数据进行...处理；
S3：根据...规则，生成...结果；
S4：输出所述结果。
```

**装置权利要求对应撰写：**
```
一种基于XXX的数据处理装置，其特征在于，包括：
接收模块，用于接收...数据；
处理模块，用于对所述数据进行...处理；
生成模块，用于根据...规则，生成...结果；
输出模块，用于输出所述结果。
```

**计算机设备/存储介质/程序产品权利要求示例：**
```
一种计算机设备，包括存储器、处理器以及存储在所述存储器上并可在所述处理器上运行的计算机程序，其特征在于，所述处理器执行所述计算机程序时实现权利要求1至X任一项所述的方法。

一种计算机可读存储介质，其上存储有计算机程序，其特征在于，所述计算机程序被处理器执行时实现权利要求1至X任一项所述的方法。

一种计算机程序产品，包括计算机程序，其特征在于，所述计算机程序被处理器执行时实现权利要求1至X任一项所述的方法。
```

## AI/算法类发明（2023年审查指南新增，2025年修订细化）

### 法律依据

《专利法》第26条第3款规定：说明书应当对发明作出**清楚、完整**的说明，以所属技术领域的技术人员**能够实现**为准。

2025年新修改的《专利审查指南》第二部分第九章进一步明确：
> "如果涉及人工智能模型的构建或者训练，则一般需要在说明书中清楚记载模型必要的**模块、层级或者连接关系**，训练必需的**具体步骤、参数**等。"

### 两类AI发明的公开要求

| 发明类型 | 必须公开的内容 | 撰写要点 |
|---------|---------------|---------|
| **AI模型的构建或训练** | ①模型架构：必要的模块、层级、连接关系；②训练过程：具体步骤、参数设置、损失函数、优化器、训练数据规模等；③推理过程：前向传播的完整数据流 | 不能只描述"采用了深度学习模型"，必须披露模型**内部结构**和**训练细节**，使技术人员能复现 |
| **AI模型在具体领域的应用** | ①模型/算法**如何与具体领域场景相结合**；②输入数据的**设置方式**（格式、维度、预处理）和**内在关联**；③输出结果的**后处理**和**应用场景映射** | 需依托"输入-处理-输出"全链路闭环，清晰披露核心技术逻辑 |

### AI模型构建/训练类发明的撰写 checklist

以下信息**至少一项缺失**即可能导致公开不充分：

- [ ] **模型架构**：网络层级数、每层的类型（卷积/全连接/注意力等）、层间连接关系、激活函数
- [ ] **输入数据**：数据格式、维度、预处理步骤（归一化/分词/编码等）
- [ ] **训练数据**：规模（样本数）、来源、标注方式、训练/验证/测试划分比例
- [ ] **损失函数**：具体公式或函数名称（如交叉熵、MSE、自定义损失）
- [ ] **优化器**：名称（SGD/Adam/AdamW等）、学习率、批次大小、训练轮数
- [ ] **关键超参数**：Dropout率、注意力头数、隐藏层维度、温度系数等
- [ ] **训练步骤**：数据加载 → 前向传播 → 损失计算 → 反向传播 → 参数更新，至少描述一轮完整的训练流程
- [ ] **推理过程**：输入 → 模型前向传播 → 输出的完整数据流

### 撰写示例（AI图像分类模型）

**❌ 差（公开不充分）：**
> 本发明采用深度学习模型对图像进行分类，模型训练后准确率可达95%以上。

**✅ 好（充分公开）：**
> 本发明采用ResNet-50卷积神经网络进行图像分类。网络包含1个初始卷积层、4个残差块组（每组分别含3/4/6/3个瓶颈残差块）和1个全连接输出层。输入图像尺寸为224×224×3，经归一化处理后输入网络。训练采用交叉熵损失函数和Adam优化器，初始学习率0.001，批次大小32，训练100轮。训练数据集包含50,000张标注图像，验证集10,000张。推理时，输入图像经相同预处理后输入训练好的模型，输出层经Softmax激活后得到各类别概率分布，取概率最大者作为分类结果。

### 软件/算法类发明的通用要求

根据《专利审查指南》第二部分第九章：
- **计算机程序本身**不属于专利保护客体
- 但**包含计算机程序的技术方案**可以：工业过程控制、计算机内部性能改善、测量/测试过程控制、外部数据处理等
- 可根据保护重点布局为方法、装置/功能模块、计算机设备/电子设备、计算机可读存储介质或计算机程序产品等权利要求
- **装置权利要求**中的组成部分应理解为**功能模块**（"接收模块""处理模块"），而非实体硬件

### 警示

**不能只做功能描述**，如"本发明利用AI模型实现智能识别，准确率高"。必须披露：
- 模型**怎么构建的**（架构）
- 模型**怎么训练的**（数据、参数、步骤）
- 模型**怎么推理的**（输入输出流程）

否则审查员会以"公开不充分"（专利法第26条第3款）或"不属于专利保护客体"为由驳回。

## 化学/生物领域

- 涉及核苷酸或氨基酸序列的，必须提交符合WIPO标准的序列表
- 化合物发明应给出明确的结构式、制备方法和至少一种用途
- 生物材料应当保藏于指定保藏机构，并在说明书中注明保藏信息
