# 分部写作详细交互模板

> 本文档是 SKILL.md「分部写作流程（逐步确认模式）」的详细展开版。每个阶段包含：输入、产出、确认要点、完整交互模板。

## 阶段1：理解技术方案

**输入：** 用户描述或技术交底书

**产出：**
1. 用自己的话复述技术方案，确认理解正确。
2. 提取核心三要素（技术问题、技术方案、有益效果）。
3. 确定专利类型（发明/实用新型）。

**输出格式（向用户展示）：**

```
📝 阶段1/6：理解技术方案

🔍 我的理解：
- 要解决的技术问题：{问题}
- 技术方案核心：{概述}
- 有益效果：{效果1/2/3}
- 建议专利类型：{发明/实用新型}

---
请确认以上理解是否正确？需要补充或修正吗？
```

**确认要点：**
- 技术方案是否准确理解？
- 有没有漏掉关键特征？
- 专利类型是否正确？
- 是否需要同日申请（发明+实用新型）？

**用户响应后：**

```bash
python scripts/draft_manager.py init <slug> <专利类型>
```

- 初始化草稿目录和 `draft.json`。
- 将用户反馈追加到 `notes.md`。

---

## 阶段2：构建权利要求

**输入：** 阶段1确认后的技术方案三要素

**产出：**
- 独立权利要求（含"其特征在于"）
- 从属权利要求（2-5 项，层次化梯度）

**撰写要求：** 遵循 SKILL.md「构建权利要求」的规则——上位概括、开放式连接词、禁用模糊用语、方法类用 S1/S2/S3。

**输出格式：**

```
📝 阶段2/6：构建权利要求

### 独立权利要求
1. 一种...，其特征在于，...

### 从属权利要求
2. 根据权利要求1所述的...，其特征在于...
3. 根据权利要求1所述的...，其特征在于...

---
✅ 已保存到 drafts/{slug}/
```

**保存草稿 + 阶段验证：**

```bash
python scripts/draft_manager.py save <slug> 2 stage2.json --notes "用户确认权利要求"
python scripts/orchestrate.py drafts/<slug>/draft.json --stage-validate 2
```

**确认要点：**
- 独立权利要求是否包含全部必要技术特征？
- 是否有非必要技术特征写入独权？
- 从属权利要求是否形成保护梯度？
- 所用术语是否与后续说明书一致？

---

## 阶段3：确定名称 + 背景技术 + 技术问题

**输入：** 阶段1理解 + 阶段2权利要求

**产出：**
- 发明名称（≤25字，禁用含糊词）
- 技术领域（1-2句）
- 背景技术（1-3段，200-400字，直击缺陷）
- 技术问题（从背景缺陷自然引出，不用"所要解决的技术问题是提供"套路）

**撰写要求：** 遵循 SKILL.md「确定发明名称」和「撰写说明书」的第1-2节。

**输出格式：**

```
📝 阶段3/6：确定名称 + 背景技术 + 技术问题

【发明名称】一种...

【技术领域】本发明涉及...技术领域，具体涉及一种...

【背景技术】
现有技术中最接近的方案及其缺陷...

【技术问题】
现有...存在...缺陷，导致...

---
请审阅。背景技术是否简练、缺陷是否准确？
```

**保存草稿 + 阶段验证：**

```bash
python scripts/draft_manager.py save <slug> 3 stage3.json --notes "用户确认背景技术和问题"
python scripts/orchestrate.py drafts/<slug>/draft.json --stage-validate 3
```

**确认要点：**
- 发明名称是否准确？是否≤25字？
- 背景技术是否直击缺陷？（无行业宏观分析/市场背景）
- 技术问题是否聚焦核心技术矛盾？（非套路化表述）
- 背景技术中的每个缺陷是否都对应一个技术问题？

---

## 阶段4：发明内容（技术方案 + 有益效果）

**输入：** 阶段2权利要求 + 阶段3背景技术/技术问题

**产出：**
- 技术方案（围绕独立权利要求展开，方法类 S1/S2/S3 分步）
- 有益效果（推导式表述"通过X→实现Y→因为Z"，量化优先）

**撰写要求：** 遵循 SKILL.md「撰写说明书」第3节的规则。

**输出格式：**

```
📝 阶段4/6：发明内容

【技术方案】
本发明提供一种...，包括：S1... S2... S3...

【有益效果】
本发明通过[区别特征]实现[效果1]，[推导逻辑]。
进一步地，[效果2]...

---
请审阅。有益效果是否有推导逻辑？是否与背景技术缺陷一一对应？
```

**保存草稿 + 阶段验证：**

```bash
python scripts/draft_manager.py save <slug> 4 stage4.json --notes "用户确认技术方案和有益效果"
python scripts/orchestrate.py drafts/<slug>/draft.json --stage-validate 4
```

**确认要点：**
- 技术方案是否紧扣独立权利要求？
- 有益效果是否有推导链？不是断言？
- 效果是否与背景技术的缺陷一一对应？

---

## 阶段5：附图说明 + 具体实施方式

**输入：** 阶段2权利要求 + 阶段4技术方案

**产出：**
- 附图说明（每图一句）
- 具体实施方式（至少一个详细实施例，S1/S2/S3 分步，含完整公式和参数定义）

**撰写要求：** 遵循 SKILL.md「撰写说明书」第4-5节的规则。

**输出格式：**

```
📝 阶段5/6：附图说明 + 具体实施方式

【附图说明】
图1是...的结构示意图
图2是...的流程图

【具体实施方式】
实施例一：
S1：...（含完整数学公式和参数定义）
S2：...

---
请审阅。实施例是否能让技术人员照着做出来？参数是否齐全？
```

**保存草稿 + 阶段验证：**

```bash
python scripts/draft_manager.py save <slug> 5 stage5.json --notes "用户确认实施例"
python scripts/orchestrate.py drafts/<slug>/draft.json --stage-validate 5
```

**确认要点：**
- 附图数量是否充足？
- 实施例是否覆盖了全部权利要求特征？
- 关键参数是否全部定义？
- 是否包含效果验证？

---

## 阶段6：撰写摘要

**输入：** 前述所有阶段的完整产出

**产出：**
- 摘要（≤300字，含名称、领域、方案要点、主要用途）

**输出格式：**

```
📝 阶段6/6：撰写摘要

{摘要正文}

---
请审阅。摘要是否≤300字？是否包含四要素？
```

**保存草稿：**

```bash
python scripts/draft_manager.py save <slug> 6 stage6.json --notes "摘要完成，进入审查"
```

**完成后** → 进入审查与生成流程（见 SKILL.md「完整输出流程 → 步骤3」）：
1. 先运行 `--dry-run` 验证无硬性错误：
   ```bash
   python scripts/orchestrate.py drafts/{slug}/draft.json --dry-run
   ```
2. 派发 4 个子代理完成语义审查
3. 保存审查报告至 reviews/
4. 运行编排脚本聚合评分并生成 Word：

```bash
python scripts/orchestrate.py drafts/{slug}/draft.json --output output.docx
```

## 用户反馈处理

每阶段确认时，用户可能的反馈及对应处理：

| 用户反馈 | 主代理处理 |
|---------|-----------|
| "可以" / "没问题" / "继续" | 确认通过，进入下一阶段 |
| "把XX改成YY" | 按指示修改，重新展示本阶段 |
| "补充一下XX" | 将补充内容纳入，重新展示 |
| "回到阶段2改一下权利要求" | 重新进入阶段2，覆盖原有产出 |
| "先停一下"/会话结束 | 草稿已保存，下次触发时自动检测 |
